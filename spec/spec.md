# Vision Mentor (视觉导师) - 互动手势匹配与 AI 评价

## 1. 核心理念与背景
原定的动作比对（DTW）技术实现复杂度较高且缺乏即时趣味性。结合“计算机视觉+非遗+交互创新”的需求，我们提出**“大师手势定格挑战 (Master Pose Match)”**方案。
用户在屏幕前模仿特定非遗技艺的经典手势（如皮影戏的“飞鸟”手影、苏绣的“捏针”、紫砂的“拍泥”），系统通过计算机视觉提取手部骨骼关键点，计算与“大师标准手势”的相似度。当相似度达到指定阈值并保持一定时间后，将触发“通关”特效，并调用大模型（LLM）生成一段结合该非遗文化背景的专属点评与鼓励。

## 2. 功能特性
- **实时手势捕捉**: 基于前端的 MediaPipe `HandTracking.jsx` 实时获取手部 21 个 3D 关键点。
- **手势相似度计算 (CV 算法)**: 后端接收骨骼数据，通过坐标归一化（平移、缩放）消除手部在画面中的位置与大小差异，计算用户手势与预置标准手势向量的余弦相似度（Cosine Similarity）或欧氏距离（Euclidean Distance），输出 0-100 的匹配度分数。
- **动态交互界面**: 
  - 前端展示“目标手势”的引导轮廓或图片。
  - 实时显示匹配度环形进度条（随分数变化颜色）。
  - 当分数 > 85 分并保持 1-2 秒后，触发“解锁成功”动画。
- **AI 智能点评**: 匹配成功后，后端调用 LLM 接口，根据当前体验的非遗项目和用户的匹配分数，生成一段生动有趣的文化点评（例如：“你的飞鸟手影灵动自如，宛如老艺人指尖跳跃的精灵！”）。

## 3. 技术架构与数据流
1. **预置数据**: 后端维护一个 `standard_poses.json`，存储各场景的标准手势特征向量（21个关键点的相对坐标）。
2. **前端定时发送**: `VisionMentor.jsx` 以约 10fps 的频率将检测到的 `landmarks` 发送至后端 `/api/v1/vision/analyze-pose`。
3. **后端比对与生成**:
   - 接收 `landmarks` 和 `scenario` (场景标识)。
   - 提取并归一化特征。
   - 计算相似度 `score`。
   - 若 `score > 85`且带有请求评价的标识，则调用 `llm_service.chat()` 生成点评 `feedback`。
   - 返回 `{"score": 90, "feedback": "...", "status": "success"}`。

## 4. 扩展性
本方案将复杂的动态序列比对简化为**静态关键帧比对**，不仅能大幅降低网络延迟与计算成本，还能轻松扩展至更多非遗项目（如剪纸手势、陶瓷拉坯手势），极大地增强了项目的互动性和可玩性。
